# Report Lifecycle Skill

## Purpose

Give any TNF agent the ability to **manage, analyze, and maintain** the JSON
report files generated by the swarm testing agents. This includes:

- **Rotation** — prevent unbounded growth of report files
- **Metadata enrichment** — tag reports with lifecycle/domain info from the
  Product Experience Architecture
- **Trend analysis** — detect declining health before it becomes critical
- **Rolling summaries** — maintain a dashboard of recent report health

## When to Use This Skill

Activate when you need to:

- Check the health/status of the testing infrastructure
- Investigate declining test scores or trends
- Clean up old report files to reclaim disk space
- Understand which Product Experience Architecture domain a report belongs to
- Generate a summary for a status update or handoff document
- Diagnose why a supercycle flagged "degraded" status

## Quick Start

### CLI Commands

```bash
# Show report inventory with counts, sizes, and metadata
tnf reports status

# Show the rolling summary dashboard
tnf reports summary

# View score trends for a specific report type
tnf reports trends test-report --limit 20
tnf reports trends integration-report
tnf reports trends uiux-report

# Prune old reports (keeps last 50 per type, removes >7 days old)
tnf reports prune

# Dry-run to see what would be pruned
tnf reports prune --dry-run

# Custom retention: keep only 20 per type, max 3 days
tnf reports prune --max-per-type 20 --max-age-days 3

# JSON output for programmatic use
tnf reports status --json
tnf reports summary --json
```

### npm Scripts

```bash
# On-demand lifecycle maintenance (rotation + summary)
pnpm reports:lifecycle

# With custom retention via env vars
REPORT_MAX_PER_TYPE=20 REPORT_MAX_AGE_MS=259200000 pnpm reports:lifecycle
```

### Programmatic API

```javascript
const {
  enrichReport,
  pruneReports,
  pruneAllReports,
  updateRollingSummary,
  writeReportWithLifecycle,
  REPORT_TYPES,
  CONFIG,
} = require('./scripts/swarm/report-lifecycle.cjs');

// Write a report with full lifecycle metadata + auto-rotation
const { filePath, pruneResult } = writeReportWithLifecycle(
  'test-report',
  reportData
);

// Enrich without writing
const enriched = enrichReport('integration-report', rawData);

// Prune a specific type
const { kept, pruned } = pruneReports('uiux-report');

// Prune everything + update summary
pruneAllReports();
updateRollingSummary();
```

## Rolling Summary

The rolling summary at `.agent/test-reports/_rolling-summary.json` tracks:

- **Average, min, max scores** per type (last 10 window)
- **Trend analysis**: `stable-or-improving` or `declining`
- **Total on-disk count** for capacity monitoring
- **Latest report reference** for quick access

### Interpreting Trends

| Trend                 | Meaning                          | Action                    |
| --------------------- | -------------------------------- | ------------------------- |
| `stable-or-improving` | Most recent score ≥ oldest score | ✅ No action needed       |
| `declining`           | Most recent score < oldest score | ⚠️ Investigate root cause |
| `insufficient-data`   | Fewer than 3 reports in window   | Wait for more data        |

All configuration via environment variables:

| `REPORT_MAX_PER_TYPE`   | `50`             | Maximum reports to keep per type   |

## Integration with Other Skills

### continuous-improvement

The rolling summary provides data for the improver agent's health assessment.
When `trend` is `declining`, the improver should investigate root causes.

### orchestrator

The supercycle flywheel calls each testing agent, which automatically triggers
report lifecycle management through `writeReportWithLifecycle()`.

### planning-with-files

Report trends can inform task planning — a declining trend may generate new
improvement tasks in `task_plan.md`.

## Troubleshooting

### Reports Not Being Pruned

1. Verify the report-lifecycle.cjs script exists:
   `ls scripts/swarm/report-lifecycle.cjs`
2. Run manually: `node scripts/swarm/report-lifecycle.cjs`
3. Check env overrides: `echo $REPORT_MAX_PER_TYPE`

### Rolling Summary Missing

1. Run `tnf reports prune` to generate it
2. Or run `pnpm reports:lifecycle`

### Metadata Not Appearing in New Reports

1. Ensure the testing agents were updated (check for `writeReportWithLifecycle`)
2. Verify: `head -5 .agent/test-reports/test-report-*.json | grep _metadata`

### Trend Shows "insufficient-data"

Need at least 3 reports in the summary window. Run the supercycle a few times:

```bash
pnpm factory:supercycle
```
