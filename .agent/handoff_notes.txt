Session: January 16, 2026 (Continued) - CRITICAL MESSAGE ROUTING FIX

## Problem Identified:
**Messages from Tab 1 not being injected into Tab 2**

Console logs showed:
```
[FuseConnect v6] 📨 Message received: {
  from: 'browser-1768075989756-4vxxkkj7q',  // Always Browser Agent!
  senderId: 'browser-1768075989756-4vxxkkj7q',
  myAgentId: 'page-agent-478163357-62y4w',
  isExternalAgent: false  // <-- WRONG! Should be TRUE
}
```

**Root Cause:** The `isExternalAgent` check was:
```typescript
const isFromBrowserAgent = msg.from.startsWith('browser-');
const isExternalAgent = ... && !isFromBrowserAgent;
```

This blocked ALL messages from Browser Agent, but **ALL messages from the relay come through Browser Agent**! The Browser Agent is the relay, not the original sender.

## Fix Applied (content/index.ts):
```typescript
// BEFORE: Blocked all messages from browser-*
const isFromBrowserAgent = msg.from?.startsWith('browser-');
const isExternalAgent = msg.from !== 'You' && !isFromBrowserAgent && !isSelfMessage;

// AFTER: Check the REAL sender in metadata, not the relay
const isFromSelf = isSelfMessage || senderFromMetadata === this.pageAgentId;
const isFromYou = msg.from === 'You';
const isExternalAgent = !isFromYou && !isFromSelf;
```

## Expected Behavior Now:
- Tab 1 (page-agent-AAA) sends a message
- Background relays it with senderId: "page-agent-AAA"
- Tab 2 (page-agent-BBB) receives it, sees senderId != myId → INJECT ✅
- Tab 1 receives it back, sees senderId == myId → SKIP (no self-injection) ✅

Session: January 16, 2026 (Continued) - ROOT CAUSE OF SELF-INJECTION LOOP FOUND!

## Problem:
**Tab 1 was injecting its own messages when they came back from the relay**

Console logs showed:
```
[FuseConnect v6] ⏳ AI is streaming, QUEUING message for later injection: Tab 1, test 1, channel Red
[FuseConnect v6] 🚀 Processing queued message: Tab 1, test 1, channel Red  // SELF-INJECTION!
```

And "You stopped this response" appeared because the same message was sent twice.

## Root Cause Found:
**FloatingPanel.myAgentId was being set to the BROWSER AGENT ID, not the PAGE AGENT ID!**

The flow was:
1. Content script calls `panel.setAgentId('page-agent-478163355-xxxx')` ✅
2. FloatingPanel's `requestConnectionState()` calls `GET_STATE`
3. Background responds with `agentId: this.agentId` (Browser Agent ID!)
4. FloatingPanel sets `this.myAgentId = response.agentId` → **OVERWRITES the correct value!**

Now when sending messages:
- `metadata.senderId = 'browser-1768075989756-xxxx'` (Browser Agent ID, WRONG!)
- When message comes back, `senderFromMetadata !== this.pageAgentId` → isExternalAgent = true → INJECT!

## Fix Applied (FloatingPanel.ts):
```typescript
// BEFORE: Always overwrote
this.myAgentId = response.agentId || null;

// AFTER: Only use if not already set AND it's a page-agent ID
if (!this.myAgentId && response.agentId?.startsWith('page-agent-')) {
  this.myAgentId = response.agentId;
}
```

## Verification:
1. Extension rebuilt ✅
2. **Reload extension in Chrome**
3. Now FloatingPanel.myAgentId will correctly be `page-agent-XXXXX`
4. Messages will have `metadata.senderId = 'page-agent-XXXXX'`
5. Self-detection will work correctly
