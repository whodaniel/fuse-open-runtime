# Security Policy

## Supported Versions

Use this section to tell people about which versions of your project are
currently being supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of **The New Fuse** seriously. If you believe you have found
a security vulnerability, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

### How to Report

If you have discovered a security vulnerability, please report it by emailing
**security@whodaniel.com**.

You can expect a response within 48 hours. If the vulnerability is accepted, we
will prioritize a fix and release a security update.

### Security Updates

We will provide security updates for the supported versions listed above. Since
this project is under active development, we recommend always running the latest
version from the `main` branch or the latest release tag.

### Migration Guide

If you are upgrading from an older version, please refer to the
[Security Migration Guide](./SECURITY_MIGRATION_GUIDE.md) for critical security
improvements and configuration requirements.
