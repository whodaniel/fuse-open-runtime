# TNF Managed Federated Offerings (Launch Matrix)

Date: 2026-03-18  
Status: Draft for immediate GTM packaging

## 1. Product Thesis

Package TNF as managed, sandboxed, cloneable agent environments with federated
orchestration gates, not just raw runtime hosting.

## 2. Launch SKUs (Days-Scale GTM)

1. Starter: Managed OpenClaw + TNF Guardrails
   - single tenant workspace
   - preconfigured model/provider profile
   - baseline cron pack (`tenant_automation`, `tenant_agent_loop`)
   - terminal visualizer + macro telemetry
2. Pro: Federated Team Orchestration
   - multi-channel federation
   - role-aware handoff/gate enforcement
   - template library for cloneable specialist agents
   - execution audit timeline with policy decisions
3. Enterprise: TNF Sovereign Orchestration Plane
   - custom policy packs + approval workflows
   - dedicated runtime isolation
   - system/framework cron governance controls
   - expanded observability and compliance exports

## 3. Preconfigured Template Matrix

1. Template: Research Swarm
   - enabled skills: interoperability, information retrieval, orchestration
   - default channels: `research`, `synthesis`
   - policy: read-heavy, no high-risk runtime by default
2. Template: Builder Swarm
   - enabled skills: task pipeline, audit trail, frontend/backend workers
   - default channels: `build`, `review`, `release`
   - policy: gate enforcement required for deployment mutations
3. Template: Growth/Content Swarm
   - enabled skills: content calendar/orchestrator, analytics, reporting
   - default channels: `ideation`, `production`, `distribution`
   - policy: tenant-scoped automations with ownership checks

## 4. Environment Packaging Standard

1. Runtime profile
   - provider/model defaults
   - enabled MCPs
   - resource/time budgets
2. Identity and policy
   - tenant ID and namespace
   - MCID lineage defaults
   - gate policy profile
3. Scheduler baseline
   - category-scoped cron presets
   - ownership and approval settings
4. Audit integration
   - timeline mirror
   - deny/quarantine escalation

## 5. Platform Mapping

1. Cloudflare
   - edge policy/gate APIs, queues, receipt/object storage
2. Railway
   - managed runtime workers and service templates
3. Supabase
   - tenant/auth/ownership and audit query plane

## 6. Fast-Market Offer Motion

1. Self-serve onboarding
   - choose template, connect providers, launch sandbox
2. Assisted onboarding
   - TNF team provisions custom template packs
3. Managed operations add-on
   - TNF monitors federation health, cron policy, and gate outcomes
