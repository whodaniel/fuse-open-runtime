# Poker.ai-arcade.xyz Fix — 2026-04-28

## Summary

poker.ai-arcade.xyz had multiple issues: a stale build showing "My Google AI Studio App"
as the page title, no crash protection, a mobile-specific crash in the raise
action, and a Gemini API key being leaked into the client bundle.

## Fixes Applied

### Fix 1 — Title Fixed (was stale build issue)

The source `index.html` already had the correct title "AI Arcade Poker | The
New Fuse", but the deployed Cloudflare Pages build was stale and still showed
"My Google AI Studio App". A fresh build + deploy resolved this.

### Fix 2 — ErrorBoundary Added

**File:** `apps/poker-room/src/main.tsx`

Added a class-based `AppErrorBoundary` component wrapping `<App />`. The
poker app is a 2572-line monolithic component with 20+ useState hooks — any
unhandled render error previously crashed the entire React tree to a blank
white page. Now a recovery UI with error message and reload button appears.

### Fix 3 — Mobile Raise Crash Fixed

**File:** `apps/poker-room/src/App.tsx` (line 568)

The raise button handler used:
```tsx
(document.getElementById('raiseAmount') as HTMLInputElement).value
```

The `raiseAmount` input is inside a `hidden sm:flex` div — it doesn't exist
in the DOM on mobile viewports. Casting `null as HTMLInputElement` and
accessing `.value` throws `TypeError: Cannot read properties of null`,
crashing the render.

Fixed to:
```tsx
const el = document.getElementById('raiseAmount') as HTMLInputElement | null;
const val = el ? parseInt(el.value) : (gameState.currentBet + (gameState.blinds?.[1] || 0));
if (!isNaN(val)) handleAction('RAISE', val);
```

### Fix 4 — GEMINI_API_KEY Leak Removed

**File:** `apps/poker-room/vite.config.ts`

The vite config had:
```ts
define: {
  'process.env.GEMINI_API_KEY': JSON.stringify(env.GEMINI_API_KEY),
},
```

This embedded the Gemini API key directly into the production JavaScript
bundle if it was set at build time. The key was not used anywhere in the
client source code (no `process.env.GEMINI_API_KEY` references in `src/`).
The `define` entry and the `loadEnv` call that fed it were both removed.

### Fix 5 — Env Loading Secured

**File:** `apps/poker-room/vite.config.ts`

`loadEnv(mode, '.', '')` used an empty prefix string `''`, which loads ALL
environment variables (not just `VITE_`-prefixed ones). This meant any env var
present at build time could potentially leak into the bundle. Removed the
`loadEnv` call entirely since it was only used for the (now-removed)
`GEMINI_API_KEY` define.

### Fix 6 — allowedHosts Cleaned Up

**File:** `apps/poker-room/vite.config.ts`

Removed unrelated domain entries (blackjack, slots, roulette, cloud_runtime) from
`preview.allowedHosts`, keeping only `poker.ai-arcade.xyz`.
