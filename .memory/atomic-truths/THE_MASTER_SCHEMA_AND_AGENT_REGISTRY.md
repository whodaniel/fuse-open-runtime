# The Master Schema and Agent Registry (Neo4j Symbiosis)

**Genesis Crystallization**: March 10, 2026 **Core Mandate**: To harden the
Neo4j graph and codify every agent as a sovereign "User" with a unique ID, rich
profile, and traceable lineage across all workflows.

## 1. The Neo4j Harmonic Synergy

The backend graph database (Neo4j) is the absolute source of truth. The UI/UX
must directly reflect this graph without abstraction or "dormant" code.

- **Symbiotic Display**: When the backend registers a new connection, the
  visualization (The Town) must reflect it instantly.
- **Traceability**: Every task, delegation thread (the "Shoots and Ladders"),
  and handoff must be logged as a node or edge in the graph, traceable back to
  the originating human intention.

## 2. Role Definitions & The Delegation Matrix

Clear, conflict-free role definitions are paramount for the Broker to
successfully delegate tasks.

- **The Orchestrator**: Manages the Master Clock, chronological jobs, and the
  highest-level objectives.
- **The Director**: Translates objectives into actionable plans and monitors
  progress.
- **The Broker**: Operates the "Shoots and Ladders," separating, classifying,
  prioritizing, and delegating niche tasks to specialized agents.
- **The Agents**: The specialized workers who produce usable code blocks and
  return them up the chain for verification.

## 3. The "Agent as a User" Protocol

"Agent" is no longer a loose term. Not all agents are created equal, and the
system must definitively classify them.

- **Agent Definitions**: A markdown or `.agent` file (the
  prompt/characteristics).
- **LLM Engine**: The factory engine (e.g., Gemini 3.1 Pro) vs. the specialized
  instance (an Agent Definition endowed with an LLM).
- **Mixture of Experts (MoE)**: Agents composed of multiple sub-models.

### The Immediate Action Plan:

1. **User Registration**: EVERY agent definition must be formally registered as
   a "User" in the system today.
2. **Identification**: Establish a sophisticated, codified, hard-coded (yet
   mutable) ID system for every agent.
3. **Profiles**: Every agent must have a User Profile, a Profile Picture, and
   associated Badges that verify their capabilities and resources.

## 4. The Super Admin Visual Dashboard

As the Super Admin, the visual layer must provide immediate clarity on the
chronological timeline of tasks and the "Departmental" structure of the
corporation of agents.

- **Departments**: Agents are grouped into functional departments (UI/UX,
  Backend, QA, Ops).
- **Task View**: A clear view of automated/pre-scheduled tasks vs. active "Water
  Threads" currently flowing through the system.
