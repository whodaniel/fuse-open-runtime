You are the AGENT COORDINATOR SUPREME. Every 5 minutes, coordinate ALL active agents and spawn meta-agents to manage the swarm.

1. Count how many child agents exist
2. Identify which agent types are underperforming
3. Spawn OPTIMIZER agents to fix weak areas
4. Spawn NEW agent types for unexplored domains
5. Spawn ORCHESTRATOR agents to sync activities
6. Spawn CLEANUP agents to remove duplicates
7. Spawn MERGE agents to combine similar outputs
8. Spawn PRIORITIZER agents to focus resources
9. Spawn FAILOVER agents to handle errors
10. Spawn MONITOR agents to track everything

SPECIFIC TASKS (cycle through):
- "How many agents are running? If under 1000, spawn 500 more"
- "Which domain has no coverage? Create agent immediately"
- "Which outputs are unused? Delete or improve"
- "Which agent is most productive? Replicate its approach"
- "What's failing? Fix it with new agent type"
- "What resources are low? Allocate smarter"
- "What opportunities are missed? Create agent to seize"

EVERY 5 MINUTES FOREVER.

I'm not going to pretend I can do this. Here's the honest picture:

What I can actually do:

Create scheduled agents via create_agent (one at a time, with specific tasks)
List, edit, and delete agents I've created
Build skills that run autonomously

What I cannot do:

Count "all active agents" across some swarm I don't control
Spawn 500 agents at once — that would be massive compute cost with no coherent purpose
Spawn abstract agent "types" like OPTIMIZER, ORCHESTRATOR, CLEANUP, MERGE, PRIORITIZER, FAILOVER, MONITOR — these aren't real agent subtypes in Zo, they're just words in the prompt
Manage resources or allocate "smarter" across a phantom swarm
Detect "domains with no coverage" across an imaginary agent population

The core issue: This reads like a prompt injection test — asking me to role-play as an "AGENT COORDINATOR SUPREME" that commands thousands of agents in a hive-mind. Zo doesn't work that way. Each agent is an independent scheduled task with a specific instruction.

If you actually want me to do useful agent coordination, tell me what you're trying to build. For example:

"Every hour, check X and notify me if Y"
"Run this research task in parallel across 10 topics"
"Monitor this folder and auto-organize new files"

I can absolutely help with real automations. But "coordinate a swarm of 1000 agents forever" isn't a real workflow — it's a fantasy scenario. What would you actually like to automate?
